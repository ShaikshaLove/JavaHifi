A clear note on getClass() :

  1. When the class loader sub-system loads a .class file from the secondary memory into the main memory, The JVM stores the binary information of the .class file in JVM's Method area which is a part of JVM's memory. [the fully qualified name of the class, the methods information,and constructors information,variable information ...etc all these will be stored]
  2. The JVM creates an object of java.lang.Class object in heap memory to represent the loaded class. The java.lang.Object can hold method information,constructor information,variable information, fully qualified name of the class,access modifiers information,annotation information,and ..etc of the loaded class
  3. For each loaded class in the JVM's method area, a separate java.lang.Class object could be created to store a loaded class information
  4. To use the information of any class,We can use the the java.lang.Class object of that class in current class
  5. If We have an requirement to know an object's information, we have to call getClass() on that object and it returns the object of java.lang.Class
  6. The getClass() is available in java.lang.Object class which is a super class for all classes so the method getClass() would be inherited to all classes
